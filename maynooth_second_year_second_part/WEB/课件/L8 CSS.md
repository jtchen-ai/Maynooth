## **模块三十：CSS 盒模型 (Box Model)**

### **1. 什么是 CSS 盒模型？**

- 所有 HTML 元素都可以被看作是盒子。
- 在 CSS 中，当我们讨论设计和布局时，会使用“盒模型”这个术语。
- CSS 盒模型本质上是一个包裹着每个 HTML 元素的盒子。
- 它由外边距 (margins)、边框 (borders)、内边距 (padding) 和实际内容 (content) 组成。

### **2. 盒模型构成**

- **Content (内容)** - 盒子的内容，文本和图片出现的地方。
- **Padding (内边距)** - 清除内容周围的一个区域。内边距是透明的。
- **Border (边框)** - 包围在内边距和内容外的边框。
- **Margin (外边距)** - 清除边框外的一个区域。外边距是透明的。
- 盒模型允许我们在元素周围添加边框，并定义元素之间的空间。

### **3. 元素的宽度和高度计算**

- 当你用 CSS 设置一个元素的 `width` 和 `height` 属性时，你只是设置了内容区域的宽度和高度。
- 要计算一个元素的完整尺寸，你还必须加上内边距、边框和外边距。
- **元素总宽度** = `width` + 左内边距 + 右内边距 + 左边框 + 右边框 + 左外边距 + 右外边距
- **元素总高度** = `height` + 上内边距 + 下内边距 + 上边框 + 下边框 + 上外边距 + 下外边距

## **模块三十一：CSS Display 与 Visibility**

### **1. `display` 属性**

- `display` 属性指定了一个元素是否显示以及如何显示。
- 每个 HTML 元素都有一个默认的 `display` 值，这取决于它是什么类型的元素。
- 大多数元素的默认 `display` 值是 `block` 或 `inline`。

#### **块级元素 (Block-level Elements)**

- 块级元素总是从一个新行开始，并占据可用的全部宽度。
- 块级元素的例子有：`<div>`, `<h1>`-`<h6>`, `<p>`, `<form>`, `<header>`, `<footer>`, `<section>`

#### **内联元素 (Inline Elements)**

- 内联元素不会从新行开始，并且只占据必要的宽度。
- 内联元素的例子有：`<span>`, `<a>`, `<img>`

### **2. 覆盖默认的 `display` 值**

- 你可以覆盖元素的默认 `display` 值。
- 将内联元素改为块级元素，或反之，对于使页面呈现特定外观同时遵循 Web 标准是很有用的。
- 一个常见的例子是为水平菜单制作内联的 `<li>` 元素。
- **注意**: 设置元素的 `display` 属性只改变元素的**显示方式**，而不是它是什么类型的元素。因此，一个设置为 `display: block` 的内联元素，其内部不允许包含其他块级元素。

### **3. `display: inline-block`**

- 与 `display: inline` 相比，`display: inline-block` 允许在元素上设置宽度和高度，并且内边距会得到保留，而 `display: inline` 则不会。
- 与 `display: block` 的主要区别是，`display: inline-block` 不会在元素后添加换行符。

### **4. 隐藏元素：`display:none` vs. `visibility:hidden`**

#### **`display: none`**

- 可以通过将 `display` 属性设置为 `none` 来隐藏一个元素。
- 元素将被隐藏，并且页面将如同该元素不存在一样显示。

#### **`visibility: hidden`**

- `visibility: hidden;` 也会隐藏一个元素。
- 然而，该元素仍然会占据和以前一样的空间。元素会被隐藏，但仍然会影响布局。

## **模块三十二：CSS 定位 (Positioning)**

### **1. `position` 属性**

- `position` 属性指定了用于元素的定位方法类型。
- 有四种不同的定位值：`static`, `relative`, `fixed`, `absolute`。
- 元素随后会使用 `top`, `bottom`, `left`, 和 `right` 属性进行定位。
- 然而，除非首先设置了 `position` 属性，否则这些属性将不起作用。它们根据 `position` 值的不同，工作方式也有所不同。

### **2. 定位类型详解**

#### **`position: static`**

- HTML 元素默认是静态定位的。
- 静态定位的元素不受 `top`, `bottom`, `left`, 和 `right` 属性的影响。
- 一个 `position: static;` 的元素不会以任何特殊方式定位；它总是根据页面的正常流来定位。

#### **`position: relative`**

- 一个 `position: relative;` 的元素是相对于其正常位置进行定位的。
- 设置相对定位元素的 `top`, `right`, `bottom`, 和 `left` 属性将导致它从其正常位置调整。
- 其他内容不会被调整以填补元素留下的任何空隙。

#### **`position: fixed`**

- 一个 `position: fixed;` 的元素是相对于视口（viewport）进行定位的，这意味着即使页面滚动，它也总停留在同一个地方。
- `top`, `right`, `bottom`, 和 `left` 属性用于定位该元素。

#### **`position: absolute`**

- 一个 `position: absolute;` 的元素是相对于其最近的**已定位祖先元素**进行定位的。
- 如果一个绝对定位的元素没有已定位的祖先，它会使用文档的 `body`，并随页面滚动而移动。
- **注意**：“已定位”的元素是指其 `position` 值除 `static` 之外的任何元素。

### **3. 在图片上定位文本**

- 你可以通过将容器元素设置为 `position: relative;` 并将内部的文本元素设置为 `position: absolute;` 来在图片上定位文本。

## **模块三十三：CSS 选择器进阶**

### **1. CSS 组合器 (Combinators)**

- 一个 CSS 选择器可以包含多个简单选择器。在简单选择器之间，我们可以包含一个组合器。

CSS3 中有四种不同的组合器：

- **后代选择器 (descendant selector)** (空格)
- **子选择器 (child selector)** (`>`)
- **相邻兄弟选择器 (adjacent sibling selector)** (`+`)
- **通用兄弟选择器 (general sibling selector)** (`~`)

#### **后代选择器（空格）**

- 匹配所有作为指定元素后代的元素。
- 示例: `div p` 选择所有 `<div>` 元素内部的 `<p>` 元素。

#### **子选择器（>）**

- 选择所有作为指定元素直接子元素的元素。
- 示例: `div > p` 选择所有作为 `<div>` 元素直接子元素的 `<p>` 元素。

#### **相邻兄弟选择器（+）**

- 选择所有作为指定元素相邻兄弟的元素。
- 兄弟元素必须有相同的父元素，“相邻”意味着“紧随其后”。
- 示例: `div + p` 选择所有紧接在 `<div>` 元素之后的 `<p>` 元素。

#### **通用兄弟选择器（~）**

- 选择所有作为指定元素兄弟的元素。
- 示例: `div ~ p` 选择所有与 `<div>` 元素同级的 `<p>` 元素。

### **2. CSS 伪类 (Pseudo-classes)**

- **定义**: 伪类用于定义元素的特殊状态。例如，它可以用于：
  - 当用户鼠标悬停在元素上时设置样式。
  - 对已访问和未访问的链接设置不同样式。
  - 当元素获得焦点时设置样式。
- **语法**: `selector:pseudo-class { property: value; }`

#### **锚点伪类**

- 链接可以以不同的方式显示，例如 `:link`, `:visited`, `:hover`, `:active`
- **注意**: 为了生效，`:hover` 必须在 CSS 定义中位于 `:link` 和 `:visited` 之后。`:active` 必须在 CSS 定义中位于 `:hover` 之后。

#### **伪类与 CSS 类结合**

- 伪类可以与 CSS 类结合使用。

#### **`:first-child` 伪类**

- 匹配作为另一个元素的第一个子元素的指定元素。
- `p:first-child` 匹配任何作为其父元素第一个子元素的 `<p>` 元素。
- `p i:first-child` 匹配所有 `<p>` 元素中的第一个 `<i>` 元素。
- `p:first-child i` 匹配所有作为第一个子元素的 `<p>` 元素中的所有 `<i>` 元素。

### **3. CSS 属性选择器**

- 可以为具有特定属性或属性值的 HTML 元素设置样式。

| 选择器语法             | 说明                                             |
| ---------------------- | ------------------------------------------------ |
| `[attribute]`          | 选择带有指定属性的元素                           |
| `[attribute="value"]`  | 选择带有指定属性和值的元素                       |
| `[attribute~="value"]` | 选择属性值包含指定单词的元素（单词由空格分隔）   |
| `[attribute            | ="value"]`                                       |
| `[attribute^="value"]` | 选择属性值以指定值开头的元素（值不必是完整单词） |
| `[attribute$="value"]` | 选择属性值以指定值结尾的元素（值不必是完整单词） |
| `[attribute*="value"]` | 选择属性值包含指定值的元素（值不必是完整单词）   |

- **表单样式化**: 属性选择器对于不使用类或 ID 来样式化表单非常有用。
