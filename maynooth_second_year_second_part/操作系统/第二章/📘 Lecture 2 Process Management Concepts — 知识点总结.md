

# 📘 Lecture 2: Process Management Concepts — 知识点总结

------

## 1. **Programs, Processes, Processors**

### 📌 Programs（程序）

- 程序是按照特定顺序执行的指令集合。
- 编译器将程序代码编译成特定处理器的机器指令（目标文件）。
- 可执行文件是将多个目标文件链接后的产物。

### 📌 Processes（进程）

- 是程序的一次执行实例，是操作系统的一种抽象。
- 包含执行上下文：程序代码、数据、堆栈、参数等。
- 操作系统管理多个进程的生命周期（创建、调度、终止等）。
- 进程是动态的，有短暂或持久的生命周期（如系统服务）。
- 操作系统本身可能由多个进程组成：内存管理、进程管理等。

### 📌 Processors（处理器）

- 执行存储在内存中的指令来运行进程。
- 多数桌面电脑只有一个 CPU，但现代 CPU 拥有多个核心（core）。
- 多核处理器可并发执行多个进程。

### 📌 Time Sharing（时间共享）

- 多个就绪进程共享 CPU 资源。
- CPU 时间被划分并分配给就绪队列中的各个进程。

### 📌 Blocked Processes（阻塞进程）

- 某些进程因等待资源（如 I/O、用户输入、其他进程）而进入阻塞状态。

------

## 2. **Representing Process Abstractions**

### 📌 Process Management（进程管理）

- 操作系统的核心任务之一：创建、控制、终止进程。
- 管理执行环境，分配和保护资源，实现进程间同步。

### 📌 Process Control Block (PCB)

- 操作系统为每个进程维护一个 PCB（进程控制块）或描述符。
- PCB 包含的信息：
  - 进程标识信息（ID 等）
  - 处理器状态（寄存器、程序计数器）
  - 控制数据（优先级、状态、队列信息等）

### 📌 PCB 的作用

- 表示进程的状态和资源分配情况。
- 支持进程调度、上下文切换等功能。

------

## 3. **Process Lifecycle（进程生命周期）**

### 📌 生命周期状态

- 就绪：等待 CPU
- 运行：正在使用 CPU
- 阻塞：等待资源
- 终止：完成或被终止

### 📌 队列机制

- PCB 可能会根据状态在多个队列之间迁移。
- 使用不同的调度算法管理队列中的进程。

------

## 4. **Communicating with the Operating System**

### 📌 系统调用（System Calls）

- 用户进程与操作系统通信的唯一合法方式。
- 通过系统调用获取服务（如硬盘访问、进程创建、IPC等）。
- 系统调用通过 **软件中断（software interrupt）** 来触发。

### 📌 执行模式（Processor Modes）

- **用户模式（User Mode）**：限制访问指令集，普通程序运行时使用。
- **内核模式（Supervisor Mode）**：操作系统运行时使用，拥有全部权限。

### 📌 Mode Switching（模式切换）

- 使用软件中断将处理器从用户模式切换到内核模式。
- 提供安全的系统调用执行环境。

### 📌 Wrapper Libraries（封装库）

- 用户进程通过系统调用的 wrapper 函数与操作系统交互。
- 检查参数正确性，调用中断指令，提高可移植性。

------

## 5. **Hardware Interrupts（硬件中断）**

### 📌 中断机制

- 硬件通过发送电信号通知处理器自己需要处理或已完成任务。
- 比轮询（Polling）更高效。

### 📌 中断控制器（Interrupt Controller）

- 管理多个设备的中断请求，将其传递给处理器。

### 📌 多任务调度中的中断

- 使用 **定时器中断（Timer Interrupt）** 实现时间片轮转。
- 定时器到期后触发中断，操作系统调度器选择新的进程运行。
- 防止单个进程长期占用 CPU。
