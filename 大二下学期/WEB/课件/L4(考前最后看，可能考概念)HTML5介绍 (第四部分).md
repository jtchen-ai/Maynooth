# **HTML5介绍 (第四部分)**

## **模块十七：HTML5 简介**

### **1. HTML5 的新特性概览**

- **简化的声明**:
  - HTML5 的 DOCTYPE 声明非常简单：`<!DOCTYPE html>`
  - 字符编码 (charset) 的声明也非常简单：`<meta charset="UTF-8">`
- **新的元素**:
  - 新的语义元素，如 `<header>`, `<footer>`, `<article>`, 和 `<section>`
  - 新的表单控件属性，如 number, date, time, calendar, 和 range
  - 新的图形元素：`<canvas>`
  - 新的多媒体元素：`<audio>` 和 `<video>`
- **新的 API**:
  - HTML 地理定位 (Geolocation)
  - HTML 拖放 (Drag and Drop)
  - HTML 本地存储 (Local Storage)
  - HTML 应用程序缓存 (Application Cache)
  - HTML Web Workers
  - HTML SSE (Server-Sent Events)

## **模块十九：理解 HTML5 语义化**

### **1. 什么是语义化元素?**

- 语义是研究语言中词语和短语含义的学科。
- 语义化元素是带有意义的元素。它能向浏览器和开发者清晰地描述其含义。
- **非语义化元素示例**: `<div>` 和 `<span>` - 它们没有说明其内容是什么。
- **语义化元素示例**: `<form>`, `<table>`, 和 `<img>` - 它们清晰地定义了其内容。

### **2. 为何使用 HTML5 语义化?**

- 在 HTML4 中，开发者使用自己喜欢的 `id` 或 `class` 名称来标记页面部分，如 `<div id="header">`, `<div id="nav">` 等。这使得搜索引擎难以准确识别网页内容。
- 使用 HTML5 的 `<header>`, `<footer>`, `<nav>`, `<section>`, `<article>` 等元素，可以让机器更容易理解内容结构。
- 语义化的 Web 允许数据在应用程序、企业和社区之间共享和重用。

### **3. 常用语义化元素详解**

- **`<section>` 元素**:
  - 定义文档中的一个区域。
  - 是内容的语义分组，通常带有一个标题。
  - 例如，一个网站主页可以分为引言、内容和联系信息等区域。
- **`<article>` 元素**:
  - 指定独立的、自包含的内容。
  - 应该可以独立于网站的其余部分进行阅读。
  - 可用于论坛帖子、博客文章、新闻报道等。
- **`<header>` 元素**:
  - 指定文档或区域的页眉。
  - 用作介绍性内容的容器。
  - 一个文档中可以有多个 `<header>` 元素。
- **`<footer>` 元素**:
  - 指定文档或区域的页脚。
  - 包含有关其所在元素的信息，通常包括作者、版权信息、使用条款链接、联系信息等。
  - 一个文档中可以有多个 `<footer>` 元素。
- **`<nav>` 元素**:
  - 定义一组导航链接。
  - 用于大块的导航链接，但并非文档中的所有链接都应放在 `<nav>` 中。
- **`<aside>` 元素**:
  - 定义与其所在内容侧面相关的内容（如侧边栏）。
  - 其内容应与周围内容相关。
- **`<figure>` 和 `<figcaption>` 元素**:
  - 将图像和其标题组合在一起。
  - `<img>` 元素定义图像，`<figcaption>` 定义标题。
- **`<main>` 元素**:
  - 指定文档的主要内容。
  - 内容应是该文档唯一的。
  - 不应包含重复内容，如侧边栏、导航链接、版权信息等。
  - 一个文档中不能有多个 `<main>` 元素，且不能是 `<article>`, `<aside>`, `<footer>`, `<header>`, 或 `<nav>` 元素的后代。
- **`<time>` 元素**:
  - 定义人类可读的日期/时间。
  - 可用机器可读的方式编码，便于搜索引擎更智能地识别。
  - `datetime` 属性用于表示机器可读的时间。
- **`<mark>` 元素**:
  - 定义被标记的文本。
  - 可用于高亮文本的某些部分。

## **模块二十：从 HTML4 迁移到 HTML5**

以下步骤展示了如何将一个典型的 HTML4 页面迁移到 HTML5：

1. **更改 Doctype**:

   - 将 HTML4 的 Doctype 更改为 HTML5：

     html

     复制

     ```
     <!DOCTYPE html>
     ```

2. **更改字符编码**:

   - 将 HTML4 的编码信息更改为 HTML5 格式：

     html

     复制

     ```
     <meta charset="UTF-8">
     ```

3. **替换 `<div>` 为语义化标签**:

   - `<div id="header">` → `<header>`
   - `<div id="footer">` → `<footer>`
   - `<div id="menu">` → `<nav>`
   - `<div id="content">` → `<section>`
   - `<div class="post">` → `<article>`

4. **更新 CSS 样式**:

   - 为新的 HTML5 语义元素创建 CSS 样式。
   - 移除不再使用的基于 `id` 或 `class` 的旧样式。

## **模块二十一：HTML5 编码规范与风格指南**

虽然 HTML5 在代码验证方面比 XHTML 更宽松，但创建自己的最佳实践、风格指南和编码约定非常重要。

### **为什么要使用编码规范?**

- 一致的风格使他人更容易理解和使用你的 HTML。
- 将来像 XML 阅读器这样的程序可能需要读取你的 HTML，因此使用接近 XHTML 的“格式良好”的语法是明智的。
- 始终保持代码风格智能、整洁、干净和格式良好。

### **编码约定**

- **文档类型**: 始终将 `<!DOCTYPE html>` 作为文档的第一行。

- **元素名大小写**: 推荐使用小写元素名，更整洁、统一。

- **关闭所有元素**: 即使 HTML5 中某些元素可省略，也推荐全部关闭。

- 关闭空元素

  : 推荐保留 

  ```
  /
  ```

  ，例如：

  html

  复制

  ```
  <meta charset="UTF-8" />
  ```

- **属性名大小写**: 推荐使用小写属性名。

- **属性值引号**: 推荐始终为属性值加引号，值中有空格时必须加。

- **图像属性**: `<img>` 应始终包含 `alt`，并指定尺寸以优化加载。

- **空格与等号**: 等号周围不加空格更简洁。

- **代码行长度**: 避免超过 80 字符，便于阅读。

- **空行与缩进**: 使用空行分隔逻辑块，缩进用 2 个空格，不用 TAB。

- 省略 `<html>`, `<body>`, `<head>`

  :

  - 尽管 HTML5 允许省略这些标签，但不推荐省略 `<html>` 和 `<body>`。
  - `<html>` 是指定页面语言的推荐位置，有助于可访问性和 SEO。
  - 省略这些标签可能会导致旧浏览器或某些解析器出错。
