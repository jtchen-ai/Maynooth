### **模块十二：HTML 实体与字符集**

#### **1. HTML 实体 (Entities)**

**用途**:

- 字符实体用于在 HTML 中显示预留字符。
- HTML 中的某些字符是预留的，例如小于号 (`<`) 或大于号 (`>`)，如果直接在文本中使用，浏览器可能会将它们与标签混淆。因此，这些预留字符必须用字符实体替换。
- 实体也可以用来显示键盘上没有的字符。

**语法**:

- 实体有两种形式：`&entity_name;` 或 `&#entity_number;`。
- 例如，要显示小于号，必须写成 `<` 或 `<`。

**名称 vs. 数字**:

- 使用实体名称的优点是更容易记住。
- 缺点是浏览器可能不支持所有的实体名称，但对数字的支持性很好。

**不间断空格 (Non-breaking Space)**:

- 这是一个在 HTML 中常用的字符实体：` `
- 它是一个不会断行到新行中的空格。
- 另一个常见用途是防止浏览器截断 HTML 页面中的多个空格。如果你在文本中输入 10 个空格，浏览器会移除其中的 9 个。要添加真正的空格，可以使用 ` ` 实体。

**注意事项**:

- 实体名称是区分大小写的。
- 使用 HTML 实体名称或十六进制数字，字符总能正确显示，这与页面使用的字符集（编码）无关。

**常用 HTML 字符实体**

| 显示结果 | 描述                 | 实体名称 | 实体编号 |
| :------- | :------------------- | :------- | :------- |
|          | non-breaking space   | ` `      | ` `      |
| <        | less than            | `<`      | `<`      |
| >        | greater than         | `>`      | `>`      |
| &        | ampersand            | `&`      | `&`      |
| ¢        | cent                 | `¢`      | `¢`      |
| £        | pound                | `£`      | `£`      |
| ¥        | yen                  | `¥`      | `¥`      |
| €        | euro                 | `€`      | `€`      |
| ©        | copyright            | `©`      | `©`      |
| ®        | registered trademark | `®`      | `®`      |

#### **2. HTML 编码 (字符集)**

**定义**: 为了正确显示 HTML 页面，Web 浏览器必须知道要使用的字符集（字符编码）。

**发展历程**:

- **ASCII**: 是第一个字符编码标准，定义了 127 个字母数字字符。
- **ANSI (Windows-1252)**: 是最初的 Windows 字符集，支持 256 个不同的字符代码。
- **ISO-8859-1**: 是 HTML 4 的默认字符集，也支持 256 个不同的字符代码。
- **UTF-8 (Unicode)**: 由于 ANSI 和 ISO 的局限性，HTML5 的默认字符编码改为了 UTF-8。UTF-8 几乎涵盖了世界上所有的字符和符号。

**如何在 HTML 中指定字符集**:

- 字符集在 `<meta>` 标签中指定。
- **HTML4**: `<meta http-equiv="Content-Type" content="text/html;charset=ISO-8859-1">`
- **HTML5**: `<meta charset="UTF-8">`

### **模块十三：HTML URL (统一资源定位符)**

**定义**:

- URL (Uniform Resource Locators) 是网页地址的另一种说法。
- Web 浏览器通过使用 URL 从 Web 服务器请求页面。
- URL 用于在 Web 上定位一个文档或其他数据。

**URL 语法**:

- URL 的语法规则如下：`scheme://prefix.domain:port/path/filename`
- **scheme**: 定义互联网服务的类型，最常见的是 `http`。
- **prefix**: 定义域名前缀，http 的默认前缀是 `www`。
- **domain**: 定义互联网域名，例如 `fzu.edu.cn`。
- **port**: 定义主机上的端口号，http 的默认端口是 80。
- **path**: 定义服务器上的路径。
- **filename**: 定义文档或资源的名称。

**常见 URL 协议 (Scheme)**

| 协议  | 全称                               | 用途                 |
| :---- | :--------------------------------- | :------------------- |
| http  | HyperText Transfer Protocol        | 普通网页，不加密。   |
| https | Secure HyperText Transfer Protocol | 安全网页，加密。     |
| ftp   | File Transfer Protocol             | 下载或上传文件。     |
| file  |                                    | 计算机上的一个文件。 |

**URL 编码**:

- URL 只能使用 ASCII 字符集通过互联网发送。
- 如果 URL 包含非 ASCII 字符，则必须进行转换。
- URL 编码将非 ASCII 字符替换为“%”后跟十六进制数字。
- URL 不能包含空格，空格通常被替换为加号 `+` 或 `%20`。

### **模块十四：HTML 表单 (Forms)**

HTML 表单用于收集用户输入。

#### **1. 表单基本结构与属性**

**`<form>` 元素**:

- 定义一个 HTML 表单。
- `<fieldset>` 元素用于将表单中的相关数据分组。
- `<legend>` 元素为 `<fieldset>` 定义标题。

**`action` 属性**:

- 定义当表单提交时要执行的动作。
- 通常，表单被提交到一个 Web 服务器上的页面进行处理。这个处理页面被称为表单处理器。

**`method` 属性**:

- 指定提交表单时使用的 HTTP 方法，通常是 `GET` 或 `POST`。
- **GET**: 如果表单提交是被动的（如搜索引擎查询）且不含敏感信息，则可以使用 GET。使用 GET 时，表单数据在页面地址中可见。GET 适用于少量数据。
- **POST**: 如果表单正在更新数据或包含敏感信息（如密码），则应使用 POST。POST 更安全，因为提交的数据在页面地址中不可见。

**`name` 属性**:

- 每个输入字段都必须有一个 `name` 属性才能被正确提交。

#### **2. 常用表单元素**

**`<input>` 元素**:

- 是最重要的表单元素。它的行为根据 `type` 属性的不同而变化。

**`<select>` 元素**:

- 定义一个下拉列表。
- `<option>` 元素定义了列表中的可选项。可以使用 `selected` 属性来预定义一个选项。

**`<textarea>` 元素**:

- 定义一个多行输入字段（文本域）。

**`<button>` 元素**:

- 定义一个可点击的按钮。

**HTML5 新增表单元素**:

- `<datalist>`: 为 `<input>` 元素指定一个预定义选项列表。用户输入时会看到一个下拉列表。
- `<keygen>`: （已弃用）指定表单中的密钥对生成器字段。
- `<output>`: 表示计算结果（例如由脚本执行的计算）。

### **模块十五：HTML 输入框类型 (Input Types)**

#### **1. 基础输入类型**

- `text`: 定义单行文本输入。
- `password`: 定义密码字段，输入的字符会被遮盖。
- `submit`: 定义用于提交表单的按钮。
- `radio`: 定义单选按钮。允许用户在一组选项中只选择一个。
- `checkbox`: 定义复选框。允许用户在一组选项中选择零个或多个。
- `button`: 定义一个可点击的按钮。

#### **2. HTML5 新增输入类型**

HTML5 增加了许多新的输入类型，如果旧版浏览器不支持，它们将表现为 `type="text"`。

- `number`: 用于应包含数值的输入字段，可以设置限制（如 `min` 和 `max`）。
- `date`: 用于应包含日期的输入字段。
- `color`: 用于应包含颜色的输入字段。
- `range`: 用于应包含一定范围内值的输入字段，通常显示为滑块。
- `month`: 允许用户选择月份和年份。
- `week`: 允许用户选择星期和年份。
- `time`: 允许用户选择时间（无时区）。
- `datetime-local`: 允许用户选择日期和时间（无时区）。
- `email`: 用于应包含电子邮件地址的输入字段，提交时可自动验证。
- `search`: 用于搜索字段。
- `url`: 用于应包含 URL 地址的输入字段。

### **模块十六：HTML 输入框属性 (Input Attributes)**

#### **1. 通用输入属性**

- `value`: 指定输入字段的初始值。
- `readonly`: 指定输入字段为只读（无法更改）。
- `disabled`: 指定输入字段被禁用。禁用的元素不可用、不可点击，且不会被提交。
- `size`: 指定输入字段的宽度（以字符为单位）。
- `maxlength`: 指定输入字段允许的最大字符数。

#### **2. HTML5 新增输入属性**

- `autocomplete`: 指定表单或输入字段是否应具有自动完成功能（`on` 或 `off`）。
- `autofocus`: 指定页面加载时，该输入元素应自动获取焦点。
- `height` 和 `width`: 仅用于 `<input type="image">`，指定图像的高度和宽度。
- `list`: 与 `<datalist>` 元素的 `id` 关联，提供预定义选项。
- `min` 和 `max`: 指定输入元素的最小值和最大值。
- `multiple`: 指定用户可以在输入元素中输入多个值，适用于 `email` 和 `file` 类型。
- `pattern`: 指定一个正则表达式，用于检查输入字段的值。
- `placeholder`: 指定一个简短的提示，描述输入字段的期望值。
- `required`: 指定在提交表单之前必须填写该输入字段。
- `step`: 指定输入字段的合法数字间隔。
